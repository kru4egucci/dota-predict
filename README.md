# Dota 2 Match Predictor

Анализ матчей Dota 2 с помощью LLM. Собирает статистику из OpenDota/Steam, отправляет в LLM, получает прогноз с вероятностями и рекомендациями по ставкам.

## Установка

```bash
cp .env.example .env
# отредактировать .env — вставить ключи
```

## Использование

### Docker (рекомендуется)

```bash
cp .env.example .env
# отредактировать .env — вставить ключи

# запуск сервера мониторинга
docker compose up -d --build

# анализ конкретного матча
docker compose run --rm dota-predict <match_id>
```

### Локально

#### 1. Анализ конкретного матча

```bash
go run cmd/main.go <match_id>
```

Где `match_id` — ID матча (можно посмотреть на дотабаффе). Работает для завершённых и текущих матчей.

#### 2. Сервер мониторинга

```bash
go run cmd/main.go server
```

Автоматический мониторинг турнирных матчей тир-1 команд:
- Поллит Steam API (`GetLiveLeagueGames`) каждые 30 секунд
- Фильтрует: только матчи где хотя бы одна команда из тир-1 списка
- Ждёт завершения драфта (все 10 героев выбраны)
- Параллельно запускает анализ и получение букмекерских коэффициентов
- Отправляет в Telegram-канал:
  - **СТАВКА** — если коэффициент букмекера >= комфортного
  - **АНАЛИТИКА** — если ставка не рекомендована, но данные полезны

#### Необходимые ключи для сервера

| Переменная | Обязательность | Описание |
|------------|---------------|----------|
| `OPENROUTER_API_KEY` | обязательно | Ключ OpenRouter для LLM |
| `STEAM_API_KEY` | обязательно | Ключ Steam Web API (для поиска лайв турнирных матчей) |
| `TELEGRAM_BOT_TOKEN` | обязательно | Токен Telegram-бота (получить через @BotFather) |
| `TELEGRAM_CHAT_ID` | обязательно | ID канала/чата для уведомлений |
| `ODDSPAPI_API_KEY` | опционально | Ключ OddsPapi для букмекерских коэффициентов |
| `OPENROUTER_MODEL` | опционально | Модель LLM (по умолчанию `anthropic/claude-sonnet-4`) |
